基于 React (ES5) 开发的、纯前端数据处理工具，专门用于对银行客服对话日志进行自动化的关键词清洗与分类。

本项目专为内网环境及受限终端设计，无需 Node.js 环境，无需 Webpack/Babel 编译，直接双击 HTML 文件即可在浏览器中运行（兼容至 IE9+）。

✨ 核心功能
🛡️ 完全离线与数据安全：所有文件解析、数据处理和逻辑运算均在本地浏览器（Client-side）完成，数据绝不上传至任何服务器，完美契合银行内网对数据隐私的高严要求。

📂 Excel/CSV 导入导出：

支持导入 .xlsx 和 .csv 格式的原始日志文件。

处理完成后，可一键下载包含“分类结果”列的新 Excel 表格。

⚙️ 可视化关键词配置：

动态编辑：内置关键词编辑器，支持用户在界面上直接添加、删除分类及关键词（如：积存金、理财、转账等）。

持久化存储：自定义配置会自动保存至浏览器本地存储（localStorage），刷新页面不丢失。

一键重置：支持一键恢复至系统默认的分类配置。

🤖 自动分类引擎：

自动扫描表格中的 对话内容 列。

基于关键词库进行模糊匹配，自动标记业务类别（如：账户问题、信用卡、理财产品等）。

生成分类统计数据，包括总条数、已分类数、未分类数及覆盖率。

📊 数据看板：实时展示处理日志与分类统计概览。

🔧 极致兼容性：

代码完全采用 ES5 语法编写（未使用 JSX）。

移除对现代构建工具的依赖，开箱即用。

🛠️ 技术栈
核心框架: React (v17/v18 Production Build), ReactDOM

数据处理: SheetJS (xlsx.full.min.js)

图表渲染: Chart.js

开发模式:原生 JavaScript (ES5) + React.createElement
