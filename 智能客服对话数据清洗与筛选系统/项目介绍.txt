基于 React 开发的、纯前端数据处理工具，专门用于对银行客服对话日志进行自动化的关键词清洗与分类。

本项目专为内网环境及受限终端设计，采用 ES5 语法编写，无需 Node.js 环境，无需 Webpack/Babel 编译构建，直接打开 HTML 文件即可在浏览器中运行（兼容至 IE9+）。

核心功能
完全离线与数据安全 所有文件解析、数据处理和逻辑运算均在本地浏览器（Client-side）完成。数据绝不上传至任何服务器，符合银行内网对数据隐私的严格要求。

Excel/CSV 格式支持 支持导入 .xlsx 和 .csv 格式的原始日志文件。处理完成后，可生成并下载包含“分类结果”列的新 Excel 表格。

可视化关键词配置

动态编辑：内置关键词编辑器，支持用户在界面上直接添加、删除分类及关键词（如：积存金、理财、转账等）。

持久化存储：自定义配置会自动保存至浏览器本地存储（localStorage），刷新页面或重启浏览器后配置不丢失。

一键重置：支持一键丢弃自定义设置，恢复至代码默认的分类配置。

自动分类引擎

自动扫描表格中的“对话内容”列。

基于关键词库进行多维度模糊匹配，自动标记业务类别。

生成详细的分类统计数据，包括总条数、已分类数、未分类数及覆盖率。

极致兼容性 代码完全采用 ES5 语法编写（未使用 JSX），移除对现代前端构建工具的依赖，开箱即用，适配旧版浏览器环境。

技术栈
核心框架: React (v17/v18 Production Build), ReactDOM

数据处理: SheetJS (xlsx.full.min.js)

开发模式: 原生 JavaScript (ES5) + React.createElement

使用方法
准备环境 确保根目录下存在 libs 文件夹，并包含核心依赖库（react, react-dom, xlsx, core-js 等）。

启动应用 直接使用浏览器双击打开 bank-filter-offline-compatible.html 文件。

上传数据 点击界面中的上传区域选择文件。 注意：源文件中必须包含列名为“对话内容”的表头。

执行分类 点击“开始分类”按钮，系统将根据当前的关键词配置进行匹配。您可以实时查看右侧的数据看板和下方的处理日志。

导出结果 点击“下载结果”按钮，获取处理后的 Excel 文件。

目录结构说明
bank-filter-offline-compatible.html: 应用入口文件

app.es5.js: 核心业务逻辑（包含 React 组件与分类算法）

libs/: 静态依赖库目录（需包含 react.production.min.js, xlsx.full.min.js 等）

注意事项
文件格式：请确保上传的文件未加密且格式正确。

列名匹配：脚本默认硬编码识别“对话内容”这一列名，如您的数据表头不同，请调整源文件表头或修改代码中的键名。
